### üî• **Core Principles**

| Concept                                 | Explanation                                                                      |
| --------------------------------------- | -------------------------------------------------------------------------------- |
| **Isolation & Control**                 | You define your own IP space (CIDR), subnets, and routing.                       |
| **Subnet Design**                       | Public (internet-facing), Private (internal), and Local (for services like RDS). |
| **Route Tables**                        | Define how traffic flows (e.g., `0.0.0.0/0` ‚Üí Internet Gateway).                 |
| **Security Groups (Stateful Firewall)** | Allow/deny inbound/outbound traffic at instance level.                           |
| **Network ACLs (Stateless Firewall)**   | Apply rules at subnet level (more granular than SGs).                            |
| **Internet Gateway (IGW)**              | Connects VPC to internet.                                                        |
| **NAT Gateway**                         | Allows private instances to access internet without public IPs.                  |
| **VPC Endpoints**                       | Access AWS services (S3, DynamoDB) privately ‚Äî no public internet.               |
| **Transit Gateway**                     | Connect multiple VPCs and on-prem networks globally.                             |
| **Flow Logs**                           | Capture network traffic for auditing and threat detection.                       |

> üõ†Ô∏è Key Resources:
>
> * `VPC` ‚Äì The virtual network
> * `Subnet` ‚Äì Logical partition of VPC
> * `Route Table` ‚Äì Defines routing paths
> * `Security Group` ‚Äì Instance-level firewall
> * `Network ACL` ‚Äì Subnet-level firewall
> * `Internet Gateway` ‚Äì Public internet access
> * `NAT Gateway` ‚Äì Private-to-public access
> * `VPC Endpoint` ‚Äì Private access to AWS services

### üë£ **Implementation Steps**

1. **Create a VPC (via Console or CLI)**\
   ‚Üí Go to **VPC Console** ‚Üí ‚ÄúCreate VPC‚Äù\
   ‚Üí Choose name, IPv4 CIDR (e.g., `10.0.0.0/16`)\
   ‚Üí Enable DNS hostnames
2. **Create Subnets (Public & Private)**\
   ‚Üí Create:
   * `public-subnet-1` (`10.0.1.0/24`) ‚Üí Public
   * `private-subnet-1` (`10.0.2.0/24`) ‚Üí Private ‚Üí Assign to AZs (e.g., `us-east-1a`, `us-east-1b`)
3. **Attach Internet Gateway (IGW)**\
   ‚Üí Create IGW ‚Üí Attach to VPC\
   ‚Üí Update route table: `0.0.0.0/0` ‚Üí IGW
4. **Set Up NAT Gateway (for Private Subnets)**\
   ‚Üí Launch NAT Gateway in `public-subnet-1`\
   ‚Üí Update private route table: `0.0.0.0/0` ‚Üí NAT Gateway
5. **Configure Security Groups**\
   ‚Üí Create:
   * `web-sg`: Allow HTTP/HTTPS (80/443) from `0.0.0.0/0`
   * `db-sg`: Allow MySQL (3306) only from `private-subnet`
6. **Create VPC Endpoints (Optional but Recommended)**\
   ‚Üí Add endpoint for **S3**, **DynamoDB**, **KMS** ‚Üí Access without public internet
7. **Enable VPC Flow Logs**\
   ‚Üí Send logs to CloudWatch ‚Üí enable GuardDuty analysis
8. **Test with Amazon Q**\
   ‚Üí Ask: _‚ÄúShow me all EC2 instances in private subnets.‚Äù_\
   ‚Üí Returns list ‚Üí verify they have no public IPs.
9. **Use AWS Transit Gateway (For Multi-VPC)**\
   ‚Üí Create TGW ‚Üí Attach VPCs ‚Üí Route traffic globally

‚úÖ **Best Practices:**

* üîê **Use VPC Endpoints** for AWS service access (no public internet)
